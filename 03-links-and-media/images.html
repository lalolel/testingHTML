HTML Images

Links and Media Lesson 2: Embedding and optimizing images in web pages

Introduction to HTML Images

Images are crucial for modern web design, enhancing user experience and communication. The <img> element is used to embed images in HTML documents.

Basic Image Syntax

<img src="image-path" alt="description" width="300" height="200">
  • src - Path to the image file (required)
  • alt - Alternative text description (required for accessibility)
  • width/height - Dimensions in pixels (optional but recommended)

Essential Image Attributes

Attribute Purpose Required Example
src Specifies image source path Yes src="images/photo.jpg"
alt Alternative text for accessibility Yes alt="Sunset over mountains"
width Width in pixels No width="300"
height Height in pixels No height="200"
title Tooltip text on hover No title="Beautiful sunset view"
loading Lazy loading behavior No loading="lazy"

Image File Formats

JPEG (.jpg/.jpeg)

Best for: Photos, complex images

Pros: Small file size, good compression

Cons: Lossy compression, no transparency

PNG (.png)

Best for: Graphics, logos, transparency

Pros: Lossless, supports transparency

Cons: Larger file sizes

SVG (.svg)

Best for: Icons, simple graphics

Pros: Scalable, small size, editable

Cons: Not suitable for photos

WebP (.webp)

Best for: Modern web optimization

Pros: Superior compression, supports transparency

Cons: Limited browser support

Responsive Images

Responsive images adapt to different screen sizes and resolutions, improving performance and user experience.

CSS Responsive Image

<style> .responsive-img { max-width: 100%; height: auto; } </style> <img src="large-image.jpg" alt="Responsive image example" class="responsive-img">

This image scales with the container width

Srcset for Different Resolutions

<img src="image-400.jpg" srcset="image-400.jpg 400w, image-800.jpg 800w, image-1200.jpg 1200w" sizes="(max-width: 600px) 400px, (max-width: 1000px) 800px, 1200px" alt="High-resolution image with multiple sources">

Browser chooses the best image size based on screen resolution

Figure and Figcaption Elements

Use <figure> and <figcaption> to semantically group images with their captions.

Image with Caption

<figure> <img src="chart.png" alt="Bar chart showing quarterly sales data" width="400" height="300"> <figcaption> Figure 1: Quarterly sales performance for 2023 </figcaption> </figure>

Multiple Images in Figure

<figure> <img src="before.jpg" alt="Garden before renovation" width="200"> <img src="after.jpg" alt="Garden after renovation" width="200"> <figcaption>Before and after garden renovation</figcaption> </figure>

Image Accessibility

✅ Alt Text Best Practices

  • Describe the content - What does the image show?
  • Consider context - How does it relate to surrounding content?
  • Be concise - Aim for 125 characters or less
  • Avoid redundancy - Don't repeat nearby text
  • Skip "image of" or "picture of" - Screen readers announce it's an image
  • Use empty alt for decorative images - alt=""

Good vs Poor Alt Text Examples

<!-- ❌ Poor alt text --> <img src="dog.jpg" alt="dog"> <img src="chart.png" alt="image"> <img src="logo.png" alt="picture of company logo"> <!-- ✅ Good alt text --> <img src="dog.jpg" alt="Golden retriever playing fetch in a park"> <img src="chart.png" alt="Sales increased 40% from Q1 to Q2 2023"> <img src="logo.png" alt="TechCorp Solutions"> <!-- ✅ Decorative image (empty alt) --> <img src="decorative-border.png" alt="">

Performance Optimization

Lazy Loading

<!-- Images load only when needed --> <img src="image1.jpg" alt="Description" loading="lazy"> <img src="image2.jpg" alt="Description" loading="lazy"> <!-- Critical images (above the fold) --> <img src="hero-image.jpg" alt="Description" loading="eager">

This image uses lazy loading to improve page performance

Image Optimization Tips

  • Choose the right format - JPEG for photos, PNG for graphics
  • Compress images - Use tools like TinyPNG or ImageOptim
  • Specify dimensions - Prevents layout shift during loading
  • Use lazy loading - For images below the fold
  • Consider WebP - Modern format with better compression
  • Optimize for mobile - Use responsive techniques

Advanced Image Techniques

Picture Element for Art Direction

<picture> <source media="(max-width: 600px)" srcset="mobile-image.jpg"> <source media="(max-width: 1200px)" srcset="tablet-image.jpg"> <img src="desktop-image.jpg" alt="Responsive image with art direction"> </picture>

Resize your browser to see different images for different screen sizes

<div class="gallery"> <img src="photo1.jpg" alt="Beach sunset"> <img src="photo2.jpg" alt="Mountain landscape"> <img src="photo3.jpg" alt="City skyline"> <img src="photo4.jpg" alt="Forest path"> </div>

Common Image Mistakes

❌ Avoid These Image Mistakes

  • Missing alt attributes - Required for accessibility
  • Poor alt text - "image" or "photo" doesn't help users
  • Huge file sizes - Compress images for web use
  • Wrong file format - Choose format based on image type
  • No dimensions specified - Causes layout shift during loading
  • Not optimizing for mobile - Images should work on all devices
  • Copyright violations - Only use images you have rights to
  • Decorative images with alt text - Use empty alt for purely decorative images

SVG Images

SVG (Scalable Vector Graphics) are perfect for icons, logos, and simple graphics that need to scale perfectly.

Inline SVG

<svg width="100" height="100" viewBox="0 0 100 100"> <circle cx="50" cy="50" r="40" fill="#4CAF50" stroke="#333" stroke-width="2"/> <text x="50" y="55" text-anchor="middle" font-family="Arial" font-size="14" fill="white">SVG</text> </svg>

This SVG scales perfectly at any size

External SVG File

<img src="icon.svg" alt="Search icon" width="24" height="24">

Introduction to HTML Links

Hyperlinks are what make the web "worldwide" - they connect pages, sites, and resources together. The <a> (anchor) element creates hyperlinks to other web pages, files, locations within the same page, email addresses, or any other URL.

Basic Link Syntax

<a href="destination">Link Text</a>
  • href - The destination URL or path
  • Link Text - The clickable text users see

External Links

External links point to resources on other domains. They should typically open in new tabs and include security attributes.

Basic External Links

<a href="https://www.mozilla.org">Visit Mozilla</a> <a href="https://developer.mozilla.org" target="_blank">MDN Docs (New Tab)</a> <a href="https://github.com" target="_blank" rel="noopener noreferrer">GitHub (Secure)</a>
Visit Mozilla
MDN Docs (New Tab)

Security Best Practice

When using target="_blank", always include rel="noopener noreferrer" to prevent security vulnerabilities and performance issues.

Link Attributes

Attribute Purpose Example
href Specifies the link destination href="https://example.com"
target Specifies where to open the link target="_blank"
rel Relationship between current and linked document rel="noopener noreferrer"
download Prompts download with optional filename download="filename.pdf"
title Additional information (tooltip) title="Learn more about HTML"
hreflang Language of the linked document hreflang="es"

Target Attribute Values

Value Description
_self Default - Opens in the same frame/tab
_blank Opens in a new window or tab
_parent Opens in the parent frame
_top Opens in the full body of the window

Accessibility Best Practices

‚úÖ Link Accessibility Guidelines

  • Use descriptive link text - Avoid "click here" or "read more"
  • Ensure sufficient color contrast - Links should be easily distinguishable
  • Provide focus indicators - Make links keyboard navigable
  • Use ARIA labels when needed - Add context for screen readers
  • Test with keyboard navigation - Ensure all links are reachable via Tab
  • Indicate external links - Let users know when leaving your site

Good vs Poor Link Text

<!-- ‚ùå Poor link text --> <p>To learn more about HTML5 features, <a href="html5-guide.html">click here</a>.</p> <p>Download our guide <a href="guide.pdf">here</a>.</p> <!-- ‚úÖ Good link text --> <p>Read our comprehensive <a href="html5-guide.html">HTML5 features guide</a>.</p> <p>Download the <a href="guide.pdf">beginner's HTML guide (PDF, 2MB)</a>.</p>

Common Link Mistakes

‚ùå Avoid These Common Mistakes

  • Broken links - Always test your links before publishing
  • Non-descriptive link text - "Click here" doesn't help users or SEO
  • Missing security attributes - Always use rel="noopener noreferrer" with target="_blank"
  • Links that don't look like links - Maintain visual consistency
  • Opening everything in new tabs - Let users control their browsing experience
  • Forgetting mobile users - Ensure links are easily tappable on mobile devices
  • Not indicating file types - Users should know what they're downloading

Forms Lesson 1: Creating interactive user input interfaces

Introduction to HTML Forms

HTML forms are used to collect user input and send it to a server for processing. Forms are essential for user interaction, from simple contact forms to complex applications.

Common Form Use Cases

  • ✅ Contact forms - Getting messages from visitors
  • ✅ Registration - Creating user accounts
  • ✅ Login - User authentication
  • ✅ Surveys - Collecting feedback and data
  • ✅ E-commerce - Shopping carts and checkout
  • ✅ Search - Finding content on websites

Form Attributes

Attribute Purpose Example
action Where to send form data action="/submit"
method HTTP method (GET/POST) method="post"
enctype Encoding type for form data enctype="multipart/form-data"
target Where to display response target="_blank"
novalidate Disable browser validation novalidate
autocomplete Enable/disable autocomplete autocomplete="off"

Label Association

Proper label association is crucial for accessibility and usability.

Different Label Methods

<!-- Method 1: for/id association (recommended) --> <label for="email-method1">Email Address:</label> <input type="email" id="email-method1" name="email1"> <!-- Method 2: nested label --> <label> Phone Number: <input type="tel" name="phone"> </label> <!-- Method 3: aria-labelledby for complex associations --> <div id="address-label">Home Address</div> <input type="text" name="address" aria-labelledby="address-label">

Form Validation

HTML5 Validation Attributes

<form> <!-- Required field --> <label for="required-field">Required Field:</label> <input type="text" id="required-field" name="required" required> <!-- Minimum and maximum length --> <label for="username-val">Username (3-20 characters):</label> <input type="text" id="username-val" name="username" minlength="3" maxlength="20" required> <!-- Pattern validation --> <label for="phone-val">Phone (XXX-XXX-XXXX):</label> <input type="tel" id="phone-val" name="phone" pattern="[0-9]{3}-[0-9]{3}-[0-9]{4}" placeholder="123-456-7890"> <!-- Number range --> <label for="rating">Rating (1-10):</label> <input type="number" id="rating" name="rating" min="1" max="10" step="1"> <button type="submit">Submit</button> </form>

Form Accessibility Best Practices

✅ Form Accessibility Guidelines

  • Always use labels - Every form control needs a label
  • Group related fields - Use fieldset and legend for groups
  • Indicate required fields - Use visual and programmatic indicators
  • Provide clear instructions - Help users understand what's expected
  • Use descriptive error messages - Tell users how to fix problems
  • Ensure keyboard navigation - All controls should be reachable via Tab
  • Test with screen readers - Verify the form works with assistive technology

Common Form Mistakes

❌ Avoid These Form Mistakes

  • Missing labels - Every input needs an associated label
  • Poor placeholder usage - Placeholders don't replace labels
  • No validation feedback - Users need to know when something's wrong
  • Unclear required fields - Mark required fields clearly
  • Complex forms without organization - Use fieldsets to group related fields
  • Auto-submit forms - Give users control over form submission
  • Too many required fields - Only require what's absolutely necessary
  • Poor mobile experience - Test forms on mobile devices

Tables Lesson 1: Creating structured data layouts

Introduction to HTML Tables

HTML tables are used to display data in rows and columns. They're perfect for presenting structured information like financial data, schedules, or comparison charts.

When to Use Tables

  • ✅ Tabular data - Information that naturally fits in rows and columns
  • ✅ Data comparison - Comparing features, prices, or statistics
  • ✅ Financial reports - Budgets, invoices, price lists
  • ❌ Page layout - Use CSS Grid or Flexbox instead
  • ❌ Simple lists - Use <ul> or <ol> elements

Table Accessibility Basics

Headers with Scope Attributes

<table> <thead> <tr> <th scope="col">Name</th> <th scope="col">Email</th> <th scope="col">Department</th> </tr> </thead> <tbody> <tr> <th scope="row">Alice Johnson</th> <td>alice@company.com</td> <td>Marketing</td> </tr> <tr> <th scope="row">Bob Smith</th> <td>bob@company.com</td> <td>Engineering</td> </tr> </tbody> </table>
Name Email Department
Alice Johnson alice@company.com Marketing
Bob Smith bob@company.com Engineering

Scope attributes help screen readers:

  • scope="col" - Header applies to the entire column
  • scope="row" - Header applies to the entire row
  • scope="colgroup" - Header applies to a group of columns
  • scope="rowgroup" - Header applies to a group of rows

Common Table Mistakes

❌ Avoid These Table Mistakes

  • Using tables for layout - Tables are for data, not page structure
  • Missing table headers - Always use <th> elements for headers
  • No semantic structure - Use thead, tbody, and tfoot appropriately
  • Poor accessibility - Missing scope attributes and captions
  • Inconsistent styling - Keep formatting consistent across cells
  • Too much data - Consider pagination or filtering for large datasets
  • Not responsive - Tables should work on mobile devices

Best Practices Summary

✅ Table Best Practices

  • Use tables for tabular data only - Not for layout
  • Include meaningful captions - Describe the table's purpose
  • Use semantic HTML - thead, tbody, th, and td appropriately
  • Add scope attributes - Help screen readers understand relationships
  • Style consistently - Use CSS for visual formatting
  • Consider mobile users - Make tables responsive or provide alternatives
  • Keep it simple - Complex tables can be hard to understand
  • Test accessibility - Use screen readers to verify usability

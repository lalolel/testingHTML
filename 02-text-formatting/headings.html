Text Formatting Lesson 1: Creating proper document structure with headings

Introduction to HTML Headings

HTML provides six levels of headings, from <h1> (most important) to <h6> (least important). Headings create a hierarchical structure that helps both users and search engines understand your content organization.

The Six Heading Levels

<h1>Main Page Title</h1> <h2>Major Section</h2> <h3>Subsection</h3> <h4>Sub-subsection</h4> <h5>Minor Heading</h5> <h6>Smallest Heading</h6>

Proper Heading Hierarchy

‚úÖ Good Heading Structure

  • Start with <h1> for the main page title
  • Use <h2> for major sections
  • Use <h3> for subsections under <h2>
  • Don't skip heading levels
  • Use headings for structure, not styling

Common Heading Mistakes

‚ùå Avoid These Mistakes

1. Skipping Heading Levels

<!-- DON'T DO THIS --> <h1>Main Title</h1> <h3>Skipped h2 - BAD!</h3> <h5>Skipped h4 - VERY BAD!</h5>

2. Using Headings for Styling

<!-- DON'T DO THIS --> <h3>This text isn't a heading, I just want it bigger</h3> <p>Regular paragraph text</p> <!-- DO THIS INSTEAD --> <p style="font-size: 1.2em; font-weight: bold;">Emphasized text</p> <p>Regular paragraph text</p>

3. Multiple H1 Elements

<!-- AVOID THIS --> <h1>Welcome to My Site</h1> <h1>About Me</h1> <!-- Should be h2 --> <h1>Contact</h1> <!-- Should be h2 -->

SEO and Accessibility Benefits

Why Proper Headings Matter

üîç Search Engine Optimization (SEO)

  • Search engines use headings to understand content structure
  • H1 tags carry more weight for keyword relevance
  • Proper hierarchy helps search engines index your content
  • Well-structured headings improve search result snippets

‚ôø Accessibility

  • Screen readers use headings to navigate content
  • Users can jump between sections using heading shortcuts
  • Proper hierarchy creates a logical reading order
  • Assistive technologies build content outlines from headings

üë• User Experience

  • Clear headings help users scan content quickly
  • Logical structure improves content comprehension
  • Consistent hierarchy sets user expectations
  • Good headings work as content anchors for linking

Best Practices Summary

‚úÖ Heading Best Practices

  1. Use only one H1 per page - Usually the main page title
  2. Don't skip levels - Go h1 ‚Üí h2 ‚Üí h3, not h1 ‚Üí h3
  3. Use headings for structure - Not for visual styling
  4. Keep headings descriptive - They should summarize the section
  5. Make headings concise - Aim for clear, brief descriptions
  6. Use CSS for styling - Don't choose heading levels based on appearance
  7. Consider your audience - Use language they'll understand
  8. Test with screen readers - Ensure logical navigation flow

Text Formatting Lesson 2: Structuring and formatting text content

HTML Paragraphs

The <p> element is used to create paragraphs of text. It's one of the most common elements in HTML and automatically adds space before and after the content.

Basic Paragraph Usage

<p>This is a paragraph of text. It contains multiple sentences that form a cohesive thought or idea.</p> <p>This is another paragraph. HTML automatically adds space between paragraphs, making the content easier to read.</p> <p>Even if you write text on multiple lines in your HTML file, it will display as a single paragraph with normal spacing.</p>

This is a paragraph of text. It contains multiple sentences that form a cohesive thought or idea.

This is another paragraph. HTML automatically adds space between paragraphs, making the content easier to read.

Even if you write text on multiple lines in your HTML file, it will display as a single paragraph with normal spacing.

Text Formatting Elements

Semantic vs Presentational Elements

HTML provides both semantic elements (that describe meaning) and presentational elements (that describe appearance). Modern web development favors semantic elements.

✅ Semantic Elements (Recommended)

  • <strong> - Important text
  • <em> - Emphasized text
  • <mark> - Highlighted text
  • <code> - Computer code
  • <kbd> - Keyboard input
  • <samp> - Sample output

❌ Presentational Elements (Avoid)

  • <b> - Bold text
  • <i> - Italic text
  • <u> - Underlined text
  • <big> - Bigger text
  • <small> - Smaller text
  • <font> - Font styling

Semantic Text Elements

Element Purpose Default Appearance Example
<strong> Important, serious, urgent text Bold Important notice
<em> Emphasized text Italic Emphasized text
<mark> Highlighted text Yellow background Highlighted text
<small> Fine print, disclaimers Smaller text Fine print text
<del> Deleted text Strikethrough Deleted text

<sup> Superscript text Higher baseline x2

The COVID-19 pandemic has significantly impacted global travel. According to recent studies, travel bookings dropped by over 90% during peak lockdown periods.

Important update: Travel restrictions are gradually being lifted as vaccination rates increase worldwide.

Preformatted Text

The <pre> element preserves spaces, line breaks, and formatting exactly as written.

<pre> function greetUser(name) { if (name) { console.log("Hello, " + name + "!"); } else { console.log("Hello, World!"); } } </pre>
function greetUser(name) {
    if (name) {
        console.log("Hello, " + name + "!");
    } else {
        console.log("Hello, World!");
    }
}

Best Practices

✅ Text Formatting Best Practices

  • Use semantic elements - Choose elements based on meaning, not appearance
  • Don't overuse formatting - Too much emphasis loses its impact
  • Use CSS for styling - Separate content from presentation
  • Consider accessibility - Screen readers understand semantic elements better
  • Be consistent - Use the same elements for the same types of content
  • Test readability - Ensure formatted text is easy to read

❌ Common Mistakes

  • Using <br> for spacing - Use CSS margins instead
  • Multiple <br> tags - Use paragraph elements instead
  • Using <b> and <i> - Use <strong> and <em> instead
  • Forgetting alt attributes - Always provide alternative text
  • Overusing emphasis - Too much bold/italic text is hard to read

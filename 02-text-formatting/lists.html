Introduction to HTML Lists

Lists are fundamental for organizing information on web pages. HTML provides three types of lists, each serving different purposes:

Unordered Lists

For items where order doesn't matter

  • Grocery items
  • Features
  • Menu options

Ordered Lists

For items where order is important

  1. First step
  2. Second step
  3. Third step

Description Lists

For term-definition pairs

HTML
Markup language
CSS
Styling language

Unordered Lists (<ul>)

Unordered lists are perfect when the sequence of items doesn't matter. Each item is marked with a bullet point by default.

Basic Unordered List

<ul> <li>Apples</li> <li>Bananas</li> <li>Oranges</li> <li>Grapes</li> </ul>
  • Apples
  • Bananas
  • Oranges
  • Grapes

Ordered lists are essential when the sequence matters, such as instructions, rankings, or step-by-step processes.

  2. Create a new file
  3. Save it as index.html

Best Practices for Lists

✅ List Best Practices

  • Choose the right list type: Use ordered lists for sequences, unordered for collections
  • Keep items parallel: Use consistent grammar and structure for list items
  • Use meaningful content: Each list item should provide value
  • Nest thoughtfully: Don't create overly deep hierarchies
  • Consider accessibility: Screen readers announce list structure to users
  • Style with CSS: Use CSS for visual customization, not HTML attributes

❌ Common List Mistakes

  • Empty list items: <li></li> - Always include content
  • Incorrect nesting: Lists must be nested inside <li> elements
  • Wrong list type: Using ordered lists when order doesn't matter
  • Inconsistent formatting: Mixing different styles within the same list
  • Overusing lists: Not every group of items needs to be a list

HTML Attributes

Lesson 3: Adding information and functionality to HTML elements

What are HTML Attributes?

Attributes provide additional information about HTML elements. They are always specified in the opening tag and usually come in name/value pairs.

Basic Attribute Syntax

<tagname attribute="value">Content</tagname>

Example:

<img src="photo.jpg" alt="A beautiful sunset" width="300">
  • src - specifies the image source
  • alt - provides alternative text
  • width - sets the image width

Global Attributes

These attributes can be used on any HTML element:

Attribute Purpose Example
id Unique identifier for an element <div id="header">
class CSS class name(s) for styling <p class="intro highlight">
style Inline CSS styling <span style="color: red;">
title Tooltip text on hover <abbr title="HyperText Markup Language">
lang Language of the element's content <span lang="es">Hola</span>
hidden Hides the element <div hidden>

Practical Examples

1. ID Attribute

Used to uniquely identify an element (should be unique on the page):

<h1 id="main-title">Welcome to My Website</h1> <p id="intro-paragraph">This is the introduction.</p>

Welcome to My Website

This is the introduction.

Use cases for ID:
  • Linking to sections: <a href="#main-title">
  • CSS styling: #main-title { color: blue; }
  • JavaScript selection: document.getElementById('main-title')

2. Class Attribute

Used to group elements for styling or scripting (can be reused):

<p class="highlight important">Important notice</p> <p class="highlight">Another highlighted paragraph</p> <span class="important">Important text</span>

Important notice

Another highlighted paragraph

Important text

3. Title Attribute

Provides additional information shown as a tooltip:

<abbr title="HyperText Markup Language">HTML</abbr> is awesome! <button title="Click to submit the form">Submit</button>
HTML is awesome!

Hover over the elements above to see tooltips!

Boolean Attributes

Some attributes don't need values - their presence indicates "true":

<input type="checkbox" checked> Pre-checked <input type="text" disabled> Disabled input <script async src="script.js"></script> <div hidden>Hidden content</div>
Pre-checked

Hidden content (you can't see this!)
Boolean attributes can be written in several ways:
  • checked (recommended)
  • checked=""
  • checked="checked"

Attribute Best Practices

❌ Common Mistakes

  • Missing quotes: <img src=photo.jpg> (should be src="photo.jpg")
  • Duplicate IDs: Having multiple elements with the same ID
  • Invalid characters: Using spaces or special characters in IDs
  • Missing alt text: Images without alt attributes

✅ Best Practices

  • Always use quotes around attribute values
  • Use lowercase for attribute names
  • Provide meaningful names for IDs and classes
  • Include alt text for all images
  • Use semantic naming (e.g., "navigation" not "blue-box")
  • Validate your HTML to catch attribute errors

Data Attributes

Custom attributes that start with "data-" for storing extra information:

<div data-user-id="123" data-role="admin">User Profile</div> <button data-action="delete" data-confirm="Are you sure?">Delete</button>
User Profile
Data attributes are useful for:
  • JavaScript functionality
  • CSS styling hooks
  • Storing configuration data
  • API integration
